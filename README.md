# DFIR Windows Acquisition Playbook

## ❓ Scenario

A Windows machine has been reported as **potentially compromised**.

Indicators may include:
- Suspicious process execution
- Unknown network connections
- Possible malware infection
- Lateral movement activity
- Data exfiltration or persistence

The system might still be:
- 🟢 Live (running)
- 🔴 Powered off

---

## 🎯 Project Overview

This repository is a **structured Digital Forensics & Incident Response (DFIR) acquisition guide** for compromised Windows systems.

It helps investigators:
- Preserve volatile and non-volatile evidence
- Follow correct acquisition order
- Use industry-standard forensic tools
- Build a complete incident timeline

---

## 🧠 Core Objectives

- Capture **volatile evidence first (RAM, live data)**
- Preserve **disk-level forensic integrity**
- Extract **Windows persistence mechanisms**
- Collect **user & attacker artifacts**
- Enable **timeline reconstruction**
